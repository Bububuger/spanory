# Spanory 功能设计规范

## 适用范围
本规范用于 Spanory 所有新增功能、行为变更、跨模块重构。目标是确保设计可评审、可验证、可交接。

## 功能设计模板（必填）

### 1. 背景
- 问题定义：当前行为有什么痛点。
- 触发场景：哪些用户/系统路径会触发。
- 约束条件：时序、兼容性、环境限制。

### 2. 目标
- 主目标（必须量化）。
- 非目标（明确不做什么）。

### 3. 功能设计
- 用户视角行为：输入、输出、失败时表现。
- 系统架构：模块职责、调用链路、状态流。
- 数据结构：关键字段、默认值、边界值。

### 4. 接口契约
- CLI 契约：命令参数、输出结构、错误码。
- Runtime Adapter 契约：输入源、归一化规则、属性语义。
- OTLP 契约：上报字段、trace/span 关系。

### 5. 兼容性
- 向后兼容策略：旧命令/旧字段是否保持。
- 迁移策略：如有 breaking change，给迁移步骤。
- 回滚策略：回滚触发条件与回滚粒度。

### 6. 验收标准
- 行为验收：对应 BDD 场景列表。
- 技术验收：`check/test/test:bdd` 必须通过。
- 文档验收：README/规范文档是否同步。

### 7. 风险评估
- 主要风险项（功能风险/数据风险/时序风险）。
- 缓解措施（监控、重试、降级、开关）。

## 变更分级与要求

### L1（低风险）
- 范围：注释、文档、非行为性重构。
- 要求：unit 相关用例通过。

### L2（中风险）
- 范围：单模块行为增强，不改公共契约。
- 要求：unit + 相关 bdd 通过。

### L3（高风险）
- 范围：跨模块行为变更、上报语义变更、插件 side-effect 调整。
- 要求：全量 `check/test/test:bdd` 通过；必须有回滚方案。

## 设计评审清单
- 是否完整填写功能设计模板。
- 是否明确接口契约与兼容性。
- 是否明确验收标准与风险。
- 是否能由未参与设计的 agent 独立实现。
