# Spanory TS 迁移 Plan（第 2 批：CLI runtime/shared + adapters）(2026-03-04)

## Goal
把 CLI runtime 关键解析链路迁移到 TypeScript 源码，同时保持现有 `.js` 运行入口不变。

## Strategy
- 源码迁移：`*.js -> *.ts`（仅 runtime/shared 与 adapters）。
- 运行兼容：通过 `tsc` 生成同路径 `.js`，继续被 `src/index.js` 与测试消费。
- 验证优先：先跑模块相关单测，再跑全量门禁。

## Scope
- `packages/cli/src/runtime/shared/*`
- `packages/cli/src/runtime/{claude,openclaw,codex}/*`
- `packages/cli/tsconfig.runtime.json`（新增）
- `packages/cli/package.json`（check/build 脚本）
- 阶段 `plan.md/todo.md`

## Tasks
### T1 基建
- 新增 CLI runtime tsconfig。
- 调整 CLI check/build 以纳入 runtime TS 检查与产物生成。

### T2 源码迁移
- 将 runtime/shared 与 adapters 改为 `.ts` 源文件。
- 生成对应 `.js` 运行文件，确保索引入口与测试路径不变。

### T3 验证
- 模块相关单测：normalize / adapters / codex proxy / codex golden。
- 全量门禁：`npm run check && npm test && npm run test:bdd`。

## Acceptance
1. `npm run --workspace @spanory/cli build`
2. `cd packages/cli && npx vitest run test/unit/normalize.spec.js test/unit/adapter.spec.js test/unit/openclaw.adapter.spec.js test/unit/codex.adapter.spec.js test/unit/codex.adapter.golden.spec.js test/unit/codex.proxy.spec.js`
3. `npm run check && npm test && npm run test:bdd`
