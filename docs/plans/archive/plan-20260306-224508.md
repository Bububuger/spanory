# Plan (2026-03-06) — 本地产物与遗留 JS 清理

## 背景
仓库里当前能看到的 `*.js` 文件全部位于本地构建生成的 `dist/` 目录中；这些文件不受 Git 跟踪，但会让工作区看起来像还有 JS 源码遗留。

## 目标
- 清理仓库内与当前提交无关的本地产物目录。
- 确认剩余 `*.js` 是否全部来自构建输出，而不是源码遗留。
- 若无真实源码遗留 JS，则不做无意义 TS 迁移，只保留结论与验收记录。

## 变更范围
- 文档流程：`plan.md`、`todo.md`
- 本地清理：仓库根 `dist/`、各 workspace `dist/`、`packages/cli/build/`

## 验收标准
1. 仓库内非 `node_modules` / 非 `.git` 路径下不再存在 `*.js` 文件。
2. 仓库内非 `node_modules` / 非 `.git` 路径下不再存在 `dist/` 目录。
3. 仓库内非 `node_modules` / 非 `.git` 路径下不再存在 `packages/cli/build/`。
4. 说明明确：当前所谓“遗留 js”实为构建产物，而非被跟踪源码。
