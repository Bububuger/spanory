# Spanory 计划：项目工作流与团队风格规范（2026-03-04）

## 目标
建立一份可执行的项目工作流规范，明确开发步骤、风格约束、文档与测试更新要求，并在项目 README 提供入口，确保新 agent 上手即按统一方式开发。

## 成功标准
- 新增独立工作流文档，覆盖：开发流程、规范、功能/修复交付要求、文档与测试更新清单。
- 在 `README.md` 提供明确链接入口。
- 标准目录文档互相串联，形成“脚手架式”上手路径。

## 范围
### In Scope
- `docs/standards/project-workflow.md`
- `docs/standards/README.md`
- `README.md`（新增工作流入口）
- `plan.md` / `todo.md`

### Out of Scope
- 业务代码改动
- 测试实现改动（仅规范要求，不新增测试代码）

## 实施阶段
1. 归档当前计划与待办，建立本阶段计划。
2. 编写项目工作流文档（完整开发闭环 + 风格规范）。
3. 更新 standards 入口与 README 链接。
4. 验收检查并收尾。
