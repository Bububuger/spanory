# Spanory 计划：OpenCode 自动读取环境与可诊断日志（2026-03-04）

## Goal
让 OpenCode 插件在 GUI/后台进程未继承 shell 环境时也能自动读取 `~/.env` 完成上报；当未上报时提供明确可查日志，降低排障成本。

## Scope
- In scope:
  - `packages/opencode-plugin/src/index.js`：自动加载 `~/.env`（仅填充未设置变量）。
  - `packages/opencode-plugin/src/index.js`：新增插件日志文件与关键事件日志（endpoint 缺失、发送失败、spool、去重跳过）。
  - `packages/cli/src/index.js`：增强 `runtime opencode plugin doctor` 输出诊断线索（日志路径/状态字段）。
  - `packages/cli/test/unit/opencode.plugin.runtime.spec.js`：新增自动读取 `.env` 与缺少 endpoint 诊断日志测试。
  - `README.md` / `docs/README_zh.md`：补充日志路径与排查命令。
  - `plan.md` / `todo.md`：阶段记录。
- Out of scope:
  - 变更 OpenCode runtime 本体触发机制。
  - 增加远端日志服务。

## Acceptance
- 未设置 `OTEL_EXPORTER_OTLP_*` 时，插件可从 `~/.env` 自动读取并尝试发送。
- 未上报时，`~/.config/opencode/state/spanory/plugin.log` 可看到明确原因。
- `spanory runtime opencode plugin doctor` 能展示日志路径与更清晰的状态线索。
- 相关单测通过：`npm run --workspace @spanory/cli test -- test/unit/opencode.plugin.runtime.spec.js`。
