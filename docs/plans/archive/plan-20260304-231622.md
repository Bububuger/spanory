# Spanory TS 迁移 Plan（第 1 批：core + otlp-core）(2026-03-04)

## Goal
在不改变运行时行为与 CLI 契约的前提下，完成 `packages/core` 与 `packages/otlp-core` 的 TypeScript 基建与首批迁移，形成可持续推进的迁移模板。

## Scope
- `package.json`
- `tsconfig.base.json`（新增）
- `packages/core/*`
- `packages/otlp-core/*`
- 与本批次直接相关的最小测试/文档更新

## Non-goals
- 不在本批次迁移 CLI 主入口与 plugin side-effect 逻辑。
- 不改变 OTLP payload 字段语义与 id 生成策略。

## Tasks
### T1 阶段切换与计划归档
- 归档旧 `plan.md/todo.md` 到 `docs/plans/archive`。
- 写入本批次执行计划与 TODO。

### T1.5 测试资产与工作流加固（全功能适用）
- 引入 OTLP 金标测试集合（same input, same output）。
- 把“所有存量/新增功能都要维护金标测试”与“每个 bug fix 都要沉淀回归 case”写入标准工作流。
- 新增缺陷回归台账，形成长期积累入口。

### T2 TS 基建（根与包级）
- 根目录引入 TypeScript 依赖与共享配置。
- 为 `core` 与 `otlp-core` 增加包级 `tsconfig`，开启严格类型检查。

### T3 otlp-core 迁移到 TS（行为不变）
- 新增 `packages/otlp-core/src/index.ts`，以类型增强方式复刻现有实现。
- 保留现有 JS 入口兼容当前 import 链路，避免影响 CLI/plugin 运行。

### T4 验证与记录
- 运行最小相关门禁（先包级 check，再 CLI OTLP 单测）。
- 在 `todo.md` 勾选完成项并记录异常与风险。

## Acceptance
1. `npm run --workspace @spanory/core check`
2. `npm run --workspace @spanory/otlp-core check`
3. `npm run --workspace @spanory/cli test -- test/unit/otlp.spec.js`
4. `cd packages/cli && npx vitest run test/unit/otlp.golden.spec.js test/unit/otlp.spec.js`
