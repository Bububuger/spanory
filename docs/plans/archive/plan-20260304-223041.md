# Spanory 计划：独立功能设计与测试基线规范（2026-03-04）

## 目标
建立可长期维护的独立规范文档（脱离阶段性 plan/todo），让其他 agent 接手时能快速上手并稳定执行。

## 成功标准
- 在 `docs/standards/` 下形成独立规范目录与入口文档。
- 完成功能设计规范：覆盖需求、架构、接口契约、变更流程、验收标准。
- 完成测试基线规范：覆盖测试分层、覆盖矩阵、门禁、回归流程、失败处理。
- 完成 agent 交接规范：新 agent 30 分钟内能按文档完成一次标准变更闭环。

## 范围
### In Scope
- `plan.md` / `todo.md`（本阶段）
- `docs/standards/README.md`
- `docs/standards/feature-design-spec.md`
- `docs/standards/test-baseline-spec.md`
- `docs/standards/agent-onboarding.md`

### Out of Scope
- 本阶段不修改业务代码实现。
- 本阶段不重构测试代码，仅定义规范与执行标准。

## 原则
1. 文档独立：不依赖某个单次迭代上下文。
2. 可执行：每条规范可映射到命令或检查动作。
3. 可交接：面向“不了解历史”的 agent。

## 实施阶段
### T0 初始化
- 归档旧 `plan.md/todo.md`，创建本阶段计划。

### T1 功能设计规范
- 输出统一功能设计模板与必填项。
- 规定接口变更、兼容性、风险评估与验收标准。

### T2 测试基线规范
- 定义 unit/bdd/集成分层责任。
- 定义覆盖矩阵维护方式与门禁标准。
- 定义回归失败处理与放行条件。

### T3 Agent 交接规范
- 规定新 agent 上手流程（读取顺序、必跑命令、提交要求）。
- 提供“从需求到交付”的标准操作清单。

### T4 收尾
- 建立 standards 入口页并串联文档。
- 完成验收检查并更新 todo。
