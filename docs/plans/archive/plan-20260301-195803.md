# Spanory Strict Last-Turn Hook Mode Plan (2026-03-01)

## Goal
提供严格“每轮仅上报新增 turn”的 Hook 模式，避免每次 Stop/SessionEnd 重发整个 session。

## Scope
- `packages/cli/src/index.js`
- `packages/cli/test/bdd/hook.integration.spec.js`
- `packages/cli/test/fixtures/claude/projects/test-project/session-f.jsonl`（新增）
- `README.md`

## Tasks
### T1 失败用例定义（TDD Red）
- 增加 BDD 用例：`--last-turn-only` 时仅导出最新 turn。
- 同一 transcript 第二次触发应 skip；追加新 turn 后应只导出新 turn。

### T2 Hook 模式实现（Green）
- 新增 `--last-turn-only` CLI 选项（`runtime claude-code hook` 与 `spanory hook`）。
- 在 hook 里只选择最新 turn 的 events 发送。
- 状态文件新增 `lastTurnId/lastTurnFingerprint` 用于 turn 级去重。

### T3 文档与回归
- README 增加 Stop Hook + `--last-turn-only` 配置示例。
- 跑目标 BDD 与全量 `npm test` 验收。
