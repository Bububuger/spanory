# Spanory vs Langfuse OTLP Parity

## Source of Truth

Parity matrix is derived from Langfuse OSS ingestion and attribute constants:

- `/tmp/langfuse/packages/shared/src/server/otel/attributes.ts`
- `/tmp/langfuse/packages/shared/src/server/otel/OtelIngestionProcessor.ts`
- `/tmp/langfuse/packages/shared/src/server/otel/ObservationTypeMapper.ts`

## Scope

Runtime parsers (`@bububuger/spanory`):

- Claude Code (`claude-code`)
- Codex (`codex`)
- OpenClaw (`openclaw`)
- OpenCode (`opencode`, plugin realtime)

Pipelines covered:

- CLI replay path: `RuntimeAdapter -> Canonical Events -> backend-langfuse -> otlp-core`
- OpenClaw plugin realtime path: `plugin hooks -> Canonical Events -> backend-langfuse -> otlp-core`
- OpenCode plugin realtime path: `plugin events -> Canonical Events -> backend-langfuse -> otlp-core`

Transport: OTLP HTTP JSON payload generated by `@bububuger/otlp-core`

Both runtimes and both ingestion paths share the same canonical event schema and langfuse backend mapping.
Parity requirements are therefore defined at canonical event level; runtime-specific gaps only apply when source
transcript/hook context lacks required data.

## Gap Policy

Current policy for “100% compatibility”:

- If Langfuse field is derivable from transcript/runtime context: Spanory must emit it.
- If field is not derivable from transcript/runtime context: mark explicit gap with reason and keep schema extension point.

This means ingestion compatibility is complete for available data; unavailable fields are tracked and non-breaking.

## Runtime Coverage Notes

- `claude-code`: mature baseline used for initial parity implementation.
- `codex`: transcript parser covers `turn/agent_command/shell_command/mcp/agent_task/tool` and supports
  notify-driven `hook --last-turn-only` incremental export; proxy hijack path can enrich request/response depth with
  full-redacted capture metadata.
- `openclaw`: follows the same event categories and attribute schema as `claude-code`
  (`turn`, `agent_command`, `shell_command`, `mcp`, `agent_task`, `tool`), so Langfuse trace richness is aligned
  for derivable fields.
- OpenClaw realtime plugin mode is supported in this phase; `export/hook/backfill` remain as补数 paths.
- `opencode`: plugin runtime 复用了 canonical normalize + langfuse backend 映射，支持 turn/tool/mcp/usage/model。
- OpenCode 当前仅支持 plugin realtime 路径（不含 export/hook/backfill CLI 补数命令）。
